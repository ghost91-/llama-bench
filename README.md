# llama-bench-harness

Benchmark [llama.cpp](https://github.com/ggml-org/llama.cpp) GGUF models to find the optimal context size for a given GPU. Targets an RTX 4070 Laptop (8 GB VRAM) + 64 GB RAM system running Arch Linux.

## Scripts

| Script | Purpose |
|--------|---------|
| `fit_bench.py` | Scan fit-params and benchmark speed at various context sizes; pick the best |
| `download_models.py` | Download GGUF files to the HuggingFace cache |
| `cleanup_model_cache.py` | Remove cached HF GGUF files not listed in `models.toml` |
| `generate_models_ini.py` | Regenerate `models.ini` from manual `models.toml` `ini` selections |
| `plot_metrics.py` | Plot quality, context, and speed trade-offs |
| `kld_extract.py` | Extract y-values from a benchmark graph image (for KLD data entry) |

Run any script with `uv run`:

```
uv run fit_bench.py unsloth/Qwen3.5-9B-GGUF:Q4_K_M
uv run fit_bench.py --vision -g qwen3.6-35b-a3b
uv run fit_bench.py --scan --rescan 7d
uv run download_models.py -g qwen3.5
uv run cleanup_model_cache.py -n
uv run generate_models_ini.py --dry-run
uv run plot_metrics.py --plot all
uv run kld_extract.py path/to/image.png
```

## How it works

`fit_bench.py` takes a model tag (`repo:quant`) and:

1. Reads the model's max context from GGUF metadata
2. Runs `llama-fit-params` at increasing context sizes to find what fits in VRAM
3. Picks the best context by two rules:
   - **Dense models**: highest context that keeps all layers on GPU (`ngl = all`)
   - **MoE models**: highest context that keeps the maximum achievable `ngl`
4. Benchmarks the chosen config with `llama-bench` (pp4096 + tg128)
5. Appends the result to `fit-bench-results.csv`

`fit_bench.py` reserves `256` MiB of VRAM for non-model use, matching the hybrid-GPU setup where the RTX 4070 Laptop dGPU stays nearly idle until workloads launch on it.

With `--vision`, the fit-target becomes `256 + mmproj size`, accounting for the VRAM cost of loading the vision encoder on GPU.

### Workflows

- No tags = all models from `models.toml`, filtered by `--provider`/`--group`
- `--scan`: scan only, write cache, skip benchmarking
- `--rescan AGE`: re-scan if `scan_ts` older than AGE (e.g. `24h`, `7d`, `30m`). Default: use cache.
- `--rebench AGE`: re-bench if `bench_ts` older than AGE. Default: always bench.
- `--scan` + `--rebench` is an error (incompatible)

## Results

- `fit-bench-results.csv`: one row per model/quant/provider/mode/ubatch combination, with columns for `ctx`, `ngl`, `ubatch`, `offload`, `pp4096_tps`, `tg128_tps` (plus stddevs), `reps`, and `bench_ts`.
- `kld-results.csv`: consolidated KLD quality measurements, one row per model/quant/provider.

Never prune `fit-bench-results.csv` or `kld-results.csv`; manage result rows manually.

## models.ini generation

`generate_models_ini.py` reads explicit `ini = [...]` entries from `models.toml`, resolves their context/fit target from benchmark results or scan cache, checks that the selected GGUF files exist on disk, and writes `~/.config/llama.cpp/models.ini`. For vision-capable models:

- Matching vision and text configs: adds `mmproj-auto = on` to the text section (mmproj fits without trade-offs)
- Differing configs: creates a separate `:vision` section with reduced context and a higher `fit-target` (`256 + mmproj size` in MiB)

## Setup

```
cd llama-bench-harness
uv sync
```

Requires `llama-fit-params`, `llama-bench`, and `llama-server` on PATH (install via your system package manager).

## Development

```
uv run ruff check .       # lint
uv run basedpyright       # type check
uv run pytest             # tests
```

External files:
- `~/.config/llama.cpp/models.ini`: llama-server presets (generated by `generate_models_ini.py`)
- `~/.cache/huggingface/hub/`: GGUF model files
